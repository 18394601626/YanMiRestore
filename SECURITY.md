# 安全策略

## 支持版本

当前仅对最新主分支和最新发布版本提供安全修复：

| 版本范围 | 安全支持 |
| --- | --- |
| `main`（最新提交） | 支持 |
| 最新 release 标签 | 支持 |
| 历史旧版本 | 不保证 |

## 漏洞报告方式

请不要在公开 Issue 中直接披露漏洞细节。  
推荐使用以下私密方式：

1. 进入仓库 `Security` 页面。
2. 选择 `Report a vulnerability` 提交私密报告。
3. 提供复现步骤、影响范围、环境信息和 PoC（如有）。

如果仓库未启用 Security Advisory，请先创建一个标题为 `安全问题（请勿公开细节）` 的 Issue，仅描述“需要私下联系维护者”，不要贴漏洞细节。

## 响应时效（目标）

- 48 小时内确认收到报告。
- 7 天内给出初步评估（影响级别、受影响版本、修复计划）。
- 高危问题优先发布修复版本，并在发布说明中标注安全修复内容。

## 披露原则

- 在修复发布前，漏洞细节默认不公开。
- 修复发布后，项目会在 Changelog 或 Release Note 中说明影响范围与升级建议。
- 对负责任披露的研究人员，项目会在发布说明中致谢（若研究人员同意）。
